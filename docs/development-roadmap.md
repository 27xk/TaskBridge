# 开发路线

本文档记录 TaskBridge 的阶段目标、当前完成情况和后续优化方向。

## 阶段 1：项目骨架

状态：已完成。

- 建立后端、Android 和 Windows 桌面端目录。
- 编写架构、API、同步和路线文档。
- 增加 MySQL、Redis 和后端 Docker Compose。
- 创建 FastAPI 基础应用。
- 创建 Android Compose 基础应用。
- 创建 Electron + Vue 3 + TypeScript 基础应用。

## 阶段 2：后端 MVP

状态：已完成。

- SQLAlchemy 模型和 Alembic 迁移。
- JWT 鉴权和 Refresh Token。
- 用户注册、登录、刷新和当前用户接口。
- 任务 CRUD、软删除和用户归属校验。
- 设备注册、列表和删除。
- 统一响应结构和异常处理。
- 后端测试覆盖核心认证、任务和设备接口。

## 阶段 3：同步与通知

状态：已完成。

- HTTP 增量拉取。
- 客户端离线变更上传。
- 任务 `version` 冲突检测。
- 同步日志 `sync_logs`。
- WebSocket 在线设备管理、心跳和断线支持。
- 同一账号下多设备通知。
- WebSocket 短期 Ticket。

## 阶段 4：Android MVP

状态：已完成。

- Room 任务表和同步队列表。
- Retrofit API 客户端。
- 本地优先 TaskRepository。
- WorkManager 后台同步。
- 登录、注册、任务列表、今日任务、任务编辑。
- 前台 WebSocket 通知。
- 系统提醒和桌面小组件。

## 阶段 5：Windows MVP

状态：已完成。

- SQLite 本地缓存。
- HTTP 同步和 WebSocket 监听。
- 登录、注册、任务列表和今日任务。
- 系统托盘。
- 桌面悬浮窗。
- 全局快捷键 `Ctrl + Alt + T`。
- 系统通知、开机自启和本地提醒。

## 阶段 6：效率增强

状态：已完成基础版本，后续可继续打磨。

- 批量完成和批量删除。
- 项目和标签管理。
- 任务模板。
- 重复任务下一次生成。
- 子清单。
- 导入导出。
- 冲突处理。
- Android 分享文本添加任务。
- Windows 备份导入导出。

## 阶段 7：性能和低配设备优化

状态：进行中。

已完成：

- 客户端本地优先，减少等待网络的 UI 阻塞。
- 小组件和悬浮窗读取本地数据库，不直接请求网络。
- 任务列表查询支持分页和限制。
- 桌面端移除不必要的 Axios 依赖，使用原生 fetch。
- 同步队列持久化，失败后延迟重试。

后续建议：

- Android 列表使用更细粒度的状态拆分，减少 Compose 重组。
- Room 与 SQLite 增加常用查询索引。
- 桌面端悬浮窗限制渲染条数，减少常驻内存。
- 增加后台同步节流和合并策略。
- 增加大批量任务导入时的分批写入。
- 建立 Android、桌面端和后端的性能基准。

## 阶段 8：发布准备

状态：待完成。

- 后端生产部署文档和 Nginx 反向代理配置。
- CI 流水线：后端测试、Android 构建、桌面端类型检查和打包。
- Android Release 签名说明。
- Windows 安装包和自动更新策略。
- 崩溃日志、错误上报和同步异常追踪。
- 数据库备份和恢复方案。

## 当前优先级建议

1. 补齐 Android 和 Windows 的完整构建验证环境。
2. 增加端到端同步测试，覆盖 Android 与 Windows 互相同步。
3. 增加冲突处理 UI 的细节说明和用户引导。
4. 增加低配设备性能基准。
5. 准备发布包、安装文档和演示数据。
