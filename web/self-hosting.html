TaskBridge

自托管部署说明

这页用于从 Web/PWA 首屏直接完成部署准备。正式对外使用时，请先准备域名、HTTPS 反向代理、强密码和明确的 Web 跨域来源。

  1. 下载项目并进入部署目录 
    git clone https://github.com/27xk/TaskBridge.git
cd TaskBridge/deploy
  2. 复制生产环境配置 
    cp .env.example .env

    Windows PowerShell 使用 Copy-Item .env.example .env。

  3. 修改关键配置 
    JWT_SECRET=replace-with-a-random-secret-at-least-32-chars
MYSQL_ROOT_PASSWORD=replace-with-a-strong-password
MYSQL_PASSWORD=replace-with-a-strong-password
DATABASE_URL=mysql+pymysql://taskbridge:replace-with-a-strong-password@mysql:3306/taskbridge
REGISTRATION_ENABLED=false
WEB_CORS_ORIGINS=https://taskbridge.example.com

    DATABASE_URL 里的密码必须和 MYSQL_PASSWORD 一致。WEB_CORS_ORIGINS 应写成真实 Web/PWA 地址，例如 https://taskbridge.example.com，不要在生产环境继续使用通配来源。

  4. 启动服务 
    docker compose -f docker-compose.release.yml up -d
  5. 检查健康状态 
    curl http://127.0.0.1:8000/ready

    /ready 返回可用后，再在客户端填写服务器根地址；如果通过 Nginx 或 Caddy 暴露 HTTPS，请填写反向代理后的域名。

  6. 创建首个账号 
    docker compose -f docker-compose.release.yml exec api python -m tools.create_user --username owner --email owner@example.com

    生产环境默认关闭公开注册。首个账号创建成功后，用这个账号在 Web、桌面端或 Android 端登录。

Self-hosting

Copy .env.example, set strong secrets, pin WEB_CORS_ORIGINS to your HTTPS Web origin, start Compose, and verify /ready before signing in.

  1. Download the project and enter the deployment directory 
    git clone https://github.com/27xk/TaskBridge.git
cd TaskBridge/deploy
  2. Copy the production environment 
    cp .env.example .env

    Windows PowerShell uses Copy-Item .env.example .env.

  3. Set production values 
    JWT_SECRET=replace-with-a-random-secret-at-least-32-chars
MYSQL_ROOT_PASSWORD=replace-with-a-strong-password
MYSQL_PASSWORD=replace-with-a-strong-password
DATABASE_URL=mysql+pymysql://taskbridge:replace-with-a-strong-password@mysql:3306/taskbridge
METRICS_TOKEN=replace-with-a-random-metrics-token
REGISTRATION_ENABLED=false
WEB_CORS_ORIGINS=https://taskbridge.example.com
  4. Start services and check readiness 
    docker compose -f docker-compose.release.yml up -d
curl http://127.0.0.1:8000/ready
  5. Create the first account 
    docker compose -f docker-compose.release.yml exec api python -m tools.create_user --username owner --email owner@example.com
